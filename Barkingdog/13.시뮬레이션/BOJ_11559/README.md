# 백준 11559번 - Puyo Puyo

[문제 보러 가기](https://www.acmicpc.net/problem/11559)

## 요약

같은 색 뿌요가 4개 이상 연결되어 있으면, 한꺼번에 없어지며 연쇄가 시작됨.

이때, 없어진 뿌요 위에 다른 뿌요가 있다면, 중력의 영향을 받아 차례대로 떨어지게 된다.

또다시 같은 색의 뿌요가 4개 이상 모이면 또 터지게 되는데, 연속해서 터질때 몇 번 연속으로 연쇄가 일어날지를 계산하자.

## 핵심 규칙

입력

- 총 12개의 줄에 필드의 정보가 주어지며, 각 줄에는 6개의 문자가 있다.
- 이때 .은 빈공간이고 .이 아닌것은 각각의 색깔의 뿌요를 나타낸다.
- R은 빨강, G는 초록, B는 파랑, P는 보라, Y는 노랑이다.
- 입력으로 주어지는 필드는 뿌요들이 전부 아래로 떨어진 뒤의 상태이다. 즉, 뿌요 아래에 빈 칸이 있는 경우는 없다.

출력

- 현재 주어진 상황에서 몇연쇄가 되는지 출력한다. 하나도 터지지 않는다면 0을 출력한다.

## 예제 입출력

입력 및 출력 예시는 [문제 링크](https://www.acmicpc.net/problem/11559) 참고.

### 문제풀이

`1. 설계`

``` text
......
......
......
......
......
......
......
......
.Y....
.YG...
RRYG..
RRYGG.
```

먼저 예제를 보고 구현을 생각해보자.

가장 아랬줄 RR과 2번째 줄 RR이 붙어있기 때문에 1연쇄가 발생

R이 제거되면서 Y 2개가 1번째 줄로 이동하게 되고, Y 4개가 붙어서 2연쇄

남은 G가 내려오면서 3 연쇄가 진행된다.

붙어있는 4개는 상하좌우를 이동하면서 확인을 해야 한다. 이걸 어떻게 판단할 것인가?

뿌요는 항상 중력에 의해 아래로 내려올 것이기 때문에, 가장 아랫줄만 보고 판단하자.

1. 전체 블록을 확인해서 .이 아닌 값 찾기
2. check함수 실행 -> 동일한 색의 뿌요를 상하좌우에서 찾아가기 -> 모두 찾고, 4개 이상인 경우 boom 4개 미만인 경우 pass
3. boom 함수 실행시, 동일한 색의 뿌요를 터트리고 combo +1
4. 그다음 남은 뿌요를 down
5. 다시 1번으로 반복, 모든 블록 확인하면 combo 출력
