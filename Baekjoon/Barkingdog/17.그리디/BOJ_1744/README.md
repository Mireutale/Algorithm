# 백준 1744번 - 수 묶기

[문제 보러 가기](https://www.acmicpc.net/problem/1744)

## 문제 요약
길이가 N인 수열이 주어졌을 때, 그 수열의 합을 구하려고 한다.
이때, 수열의 두 수를 묶는다.

위치에 상관없이 묶을 수 있고, 어떤 수를 묶게 되면, 수열의 합을 구할 때 묶은 수는 서로 곱한 후에 더한다.

자기 자신은 묶을 수 없다.

모든 수는 단 한번만 묶거나, 아니면 묶지 않아야 한다.

수열의 각 수를 적절히 묶었을 때, 그 합이 최대가 되게 하는 프로그램을 작성하시오.

## 입력
- 첫째 줄에 수열의 크기 N이 주어진다. 
- N은 50보다 작은 자연수이다. 
- 둘째 줄부터 N개의 줄에 수열의 각 수가 주어진다.
- 수열의 수는 -1,000보다 크거나 같고, 1,000보다 작거나 같은 정수이다.


## 출력
- 수를 합이 최대가 나오게 묶었을 때 합을 출력한다. 
- 정답은 항상 231보다 작다.



## 예제 입력 및 출력
링크의 예제 입력 및 출력 확인

## 문제풀이

`1.설계`

수열이 {0, 1, 2, 4, 3, 5} 일때 적절하게 묶으면?
0 + 1 + (2 * 3) + (4 * 5) = 27이 최대가 된다.

예시 1번을 확인해 보면 주어진 수열은 {-1, 2, 1, 3}이다.

음수의 경우에는 다른 수와 곱하지 않거나, 0과 곱하는게 제일 좋다.

양수의 경우에는 큰 수와 묶어 곱하는게 좋다.

-1과 1은 그냥 더하고, 2와 3을 묶으면 최대 값 6이 나온다.

예시 2번을 보면 주어진 수열은 {0, 1, 2, 4, 3, 5}이다.
0과 1은 그냥 두고, 2 * 3 , 4 * 5로 묶어서 최댓값을 구할 수 있다.

`2.내림차순`

내림차순으로 정렬하고, 두개의 값을 곱해가면서 최댓값을 구하자 함

문제점
1. 양수는 가능하나, 1의 경우는 그냥 더하는게 더 이득이고 음수의 경우에는 오히려 내림차순으로 정렬해서 두 값씩 곱하는게 더 크다.
2. 0의 경우에는 그냥 더하거나, 음수가 1개만 남은 경우에 곱해서 처리하는게 제일 크다.

이렇게 종류별로 다르게 풀어야 하기 때문에, 양수, 1, 0, 음수로 나눠서 풀고자 하였다.

`3.분리 풀이`

1. 양수의 경우는 모든 양수를 모으고, 내림차순을 한 다음, 두개의 숫자끼리 묶어가면서 계산한다.
2. 1의 경우는 그냥 더한다.
3. 음수의 경우는 오름차순을 한 다음, 두개의 숫자끼리 묶어가면서 계산한다.
   1. 음수의 개수가 홀수인 경우 1개의 음수가 남게 되는데, 0이 있다면 0과 묶고 아닌 경우는 그냥 더한다.
4. 남은 0은 그냥 더한다.
