# 49994 방문 길이

## 문제 설명

4가지 명령어를 통해 캐릭터를 움직임.
- U: 위쪽으로 한 칸 가기
- D: 아래쪽으로 한 칸 가기
- R: 오른쪽으로 한 칸 가기
- L: 왼쪽으로 한 칸 가기

- 좌표명면 0,0에서 시작하며, 경계는 (5, 5), (-5, 5), (-5, -5), (5, -5)로 이루어져 있음.
- 만약, 좌표평면의 경계를 넘어가는 명령어는 무시할 것.

- 캐릭터가 지나간 길 중 **처음 걸어본 길의 길이**를 구할 것.

명령어는 매개변수 `dirs`로 주어질 때, 처음 걸어본 길의 길이를 구하여 return 하는 solution 함수를 완성

## 제한 조건
- dirs는 string 형으로 주어짐, U, D, R, L외의 문자는 주어지지 않음
- dirs의 길이는 500이하의 자연수

## 입출력 예시
1. 
   - dirs	 = "ULURRDLLU"
   - answer = 7

2. 
   - dirs = "LULLLLLLU"	
   - answer = 7

### 문제 풀이

`1. 설계`
- 처음 지나간 길의 길이를 구해야 하므로, 반드시 이전에 지나간 길을 저장하고 있어야 함.
  - 저장하는 방식을 어떻게 할지가 관건
  - (x, y, dirs) 형태 또는 (x0, y0, x1, y1)으로 이동한 좌표 형태를 가질지를 생각.
    - 문제는 위 형태가 값을 찾는데 오래 걸린다는 점.
    - dirs를 0, 1, 2, 3으로 구분
    - 3차원 배열을 구성해서 visited[x][y][dirs]의 값을 확인 -> True이면 이미 지나간 길이므로 확인하지 않는 형식

`2. 틀림`
- 잘못된 문제 풀이
  - 이미 이동한 방향의 경우, 반대쪽에서 이동하는 경우도 제한해야 함.
  - 즉 visitied[x][y][dir]에 의해서 nx, ny로 이동한 경우에는 visited[nx][ny][dir의 반대]도 해주어야 한다는 것
  - UDRL을 순서대로 0, 1, 2, 3이라고 잡았으므로 0의 반대는 1 / 2의 반대는 3이 되어야 함

- 조합 set을 활용했다라면? -> 이미 set에 있는 것은 중복을 허용하지 않아 제한될것이기 때문에, path의 개수를 세도 되었을 거 같음
